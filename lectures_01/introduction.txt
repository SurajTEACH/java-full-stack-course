Why Java ?
==============

c/c++ me portability nhi hai 

portability hota ky hai =>  c/c++ ke source code ko run kaene ke liy compiler     hota hai  jo ki souce code ko comple karta hai 

spose karo ki mere laptop me c/c++ ke code ko run karna chahte h to complile karnaa pdta hai or usi source code ko ager hm dusare computer/platfrom me run karate h to dubara complie karna padega  yehi resion h ki c/c++ me portability nhi hai 

c/c++ platfrom dependent langauge =>  agar apke pass c/c++ ke souce code hai usko agar aap complile kaena chahte ho to  har ek platfrom ke liy alg se compile karna hoga 


platfrom =>
          |__ processer
          |
          |__ os
          |

          processor  +     os    =>   platfrom

          intelx86   +     window  =>  p1 

          ARM       +      MAC     =>  p2


          agar koi c/c++ ka source code jo h usko complie karne ke liy alagr algr se complie karna hoga like p1 ke liy alg compile and p2 ke liy alg compile 

          yehi to platfrom dependent langauge hai 

          example se samgho ki ek p1 platfrom  me OS  window hai or ek platfrom p2 me MAC hai jb c/c++ ka code ko compile karega to window ke liy alg se machine code ganarate karega and  MAC ke liy alg se machine code ganarate karega   

          isiliy isko c/c++ ko platfrom dependent langauge kahate hai 


        program  => instruction set architecture (ISA)  =>  processor
